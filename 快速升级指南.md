# GoProx 快速升级指南

## 一键升级步骤

### 1. 备份配置
```bash
cp config.yaml config.yaml.backup.$(date +%Y%m%d)
```

### 2. 停止旧服务
```bash
# 查找进程
ps aux | grep goprox

# 停止服务
pkill goprox
# 或
kill $(pgrep goprox)
```

### 3. 替换二进制文件
```bash
# 备份旧版本
mv goprox goprox.old

# 编译新版本
go build -o goprox

# 或直接使用编译好的
cp /path/to/new/goprox ./goprox
chmod +x goprox
```

### 4. 启动新服务
```bash
# 前台运行（测试）
./goprox -config config.yaml

# 后台运行（生产）
nohup ./goprox -config config.yaml > goprox.log 2>&1 &
```

### 5. 验证功能
```bash
# 检查进程
ps aux | grep goprox

# 检查日志
tail -f goprox.log

# 访问测试
curl http://localhost:8080/
```

## 主要变化

### ✅ 无需修改配置文件

旧配置文件完全兼容，无需修改即可使用。

### 🆕 可选新功能

如果需要使用 `backend_path` 功能，可以添加到服务配置：

```yaml
services:
  - id: my-app
    name: "我的应用"
    host: "127.0.0.1"
    port: 8080
    path: "/myapp"
    backend_path: "/static/app/index.html"  # 新增字段（可选）
    websocket: false
```

1. **Session 自动续期** - 活跃用户不会闪回登录页
2. **多服务并发** - 可以同时使用多个服务
3. **WebSocket 验证** - 严格按配置验证
4. **加载反馈** - 点击服务卡片有加载动画

## 常见问题

### Q1: 升级后需要重新登录吗？
**A:** 可能需要。旧的 session 在升级后会保留，但建议清除浏览器 cookie 后重新登录以获得最佳体验。

### Q2: 已有的服务配置需要修改吗？
**A:** 不需要。所有现有配置完全兼容。

### Q3: backend_path 是必须的吗？
**A:** 不是。这是一个可选功能，只在需要代理到特定路径时使用。

### Q4: 如何回滚到旧版本？
**A:** 
```bash
pkill goprox
mv goprox.old goprox
./goprox -config config.yaml
```

### Q5: 升级后发现问题怎么办？
**A:** 
1. 查看日志：`tail -f goprox.log`
2. 检查配置：`cat config.yaml`
3. 回滚到旧版本
4. 报告问题并附上日志

## 生产环境建议

### 使用 systemd 管理

创建 `/etc/systemd/system/goprox.service`:

```ini
[Unit]
Description=GoProx Reverse Proxy
After=network.target

[Service]
Type=simple
User=goprox
WorkingDirectory=/opt/goprox
ExecStart=/opt/goprox/goprox -config /opt/goprox/config.yaml
Restart=always
RestartSec=10
StandardOutput=append:/var/log/goprox/goprox.log
StandardError=append:/var/log/goprox/goprox.log

[Install]
WantedBy=multi-user.target
```

启动命令：
```bash
sudo systemctl daemon-reload
sudo systemctl enable goprox
sudo systemctl start goprox
sudo systemctl status goprox
```

### 日志轮转

创建 `/etc/logrotate.d/goprox`:

```
/var/log/goprox/*.log {
    daily
    rotate 7
    compress
    delaycompress
    missingok
    notifempty
    create 0640 goprox goprox
    sharedscripts
    postrotate
        systemctl reload goprox > /dev/null 2>&1 || true
    endscript
}
```
