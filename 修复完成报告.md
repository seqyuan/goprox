# GoProx 闪回问题修复完成报告

修复时间：2026-07-16

## ✅ 已完成的修复

### P0 级别（严重问题）

#### 1. Session Cookie 滑动窗口刷新机制 ✅

**问题**：Session 固定 24 小时过期，无法续期，用户必定闪回登录页

**修复**：
- 实现滑动窗口机制：每次请求刷新 session
- 活跃用户的 session 永不过期
- 添加 `RefreshSession()` 函数自动续期
- 在所有需要认证的请求中调用刷新

**文件修改**：
- `internal/auth/session.go` - 添加 `RefreshSession()` 和 `RefreshSessionCookie()`
- `internal/proxy/proxy.go` - 在代理请求时刷新 session
- `internal/api/api.go` - 在 API 请求时刷新 session
- `internal/web/web.go` - 在页面请求时刷新 session

#### 2. Route Cookie 路径冲突修复 ✅

**问题**：单一 route cookie 在多服务切换时被覆盖

**修复**：
- 为每个服务的 cookie 设置独立路径范围
- Cookie 路径限定为 `Path=/service-path`
- 不同服务的 cookie 互不干扰

**文件修改**：
- `internal/proxy/proxy.go` - `SetRouteCookie()` 添加路径参数
- `internal/web/web.go` - 传递服务路径给 cookie

#### 3. Referer 头依赖重构 ✅

**问题**：新标签页、后退、隐私模式下 Referer 丢失

**修复**：
- 改用 route cookie 作为主要上下文
- Referer 仅作为备用
- 登录后优先使用 cookie 中的路由信息

**文件修改**：
- `internal/web/web.go` - `inferBackRoute()` 优先使用 cookie

### P1 级别（高优先级）

#### 4. WebSocket 验证和支持 ✅

**问题**：配置 `websocket: false` 不生效，缺少验证

**修复**：
- 在 WebSocket 升级前验证服务配置
- 非 WebSocket 服务拒绝升级请求
- 添加专用的 `isWebSocketUpgrade()` 函数
- 改进错误响应

**文件修改**：
- `internal/proxy/proxy.go` - 添加 WebSocket 验证逻辑

#### 5. API 路由冲突解决 ✅

**问题**：`/api/*` 与后端 SPA 应用的 API 路由冲突

**修复**：
- 精确匹配 API 路由而不是前缀匹配
- 只处理明确的管理 API 端点
- 其他 `/api/*` 请求透传给后端服务

**文件修改**：
- `internal/api/api.go` - 改为精确路由匹配

#### 6. 前端加载状态优化 ✅

**问题**：点击卡片无反馈，服务慢启动时用户困惑

**修复**：
- 添加点击后的加载动画
- 显示"正在启动服务..."提示
- 健康检查失败时显示错误
- 改进用户体验

**文件修改**：
- `internal/web/templates/dashboard.html` - 添加加载状态 UI
- `internal/web/templates/script.js` - 添加加载逻辑
- `internal/web/templates/style.css` - 添加加载动画样式

### 额外功能

#### 7. BackendPath 支持 ✅

**问题**：无法代理到后端服务的具体文件或子路径

**新功能**：
- 添加 `backend_path` 配置字段
- 支持指定后端服务的实际路径
- 适用于静态文件、嵌套应用等场景

**文件修改**：
- `internal/config/config.go` - 添加 `BackendPath` 字段
- `internal/proxy/proxy.go` - 实现路径重写逻辑
- `internal/api/api.go` - 支持 API 操作
- `internal/config/user.go` - 支持配置读写
- `internal/web/templates/dashboard.html` - 添加 UI 字段
- `internal/web/templates/script.js` - 支持表单提交

**使用示例**：
```yaml
services:
  - name: "静态站点"
    host: "127.0.0.1"
    port: 8080
    path: "/myapp"
    backend_path: "/static/index.html"  # 新增字段
```

## 🎯 核心改进

### 1. Session 生命周期管理
- **之前**：固定 24 小时后强制登出
- **现在**：活跃用户永不过期，自动续期

- **现在**：严格按配置验证

### 5. API 路由透明度
- **之前**：劫持所有 `/api/*` 请求
- **现在**：只处理管理 API，其他透传

### 6. 用户体验
- **之前**：点击无反馈，错误无提示
- **现在**：完整的加载状态和错误反馈

## 📝 技术细节

### Session 刷新机制

```go
// 每次请求都刷新 session
func RefreshSession(session *Session, secret string) string {
    now := time.Now()
    session.IssuedAt = now
    session.ExpiresAt = now.Add(24 * time.Hour)
    return session.Encode(secret)
}
```

### Cookie 路径范围

```go
// 每个服务独立的 cookie 路径
func SetRouteCookie(w http.ResponseWriter, serviceID string, servicePath string) {
    http.SetCookie(w, &http.Cookie{
        Name:     "route",
        Value:    serviceID,
        Path:     servicePath,  // 关键：限定路径范围
        HttpOnly: true,
        Secure:   false,
        SameSite: http.SameSiteLaxMode,
        MaxAge:   86400,
    })
}
```

### WebSocket 验证

```go
// 升级前检查配置
if isWebSocketUpgrade(r) {
    if !svc.WebSocket {
        http.Error(w, "WebSocket not enabled for this service", 
                   http.StatusBadRequest)
        return
    }
}
```

### 路径重写逻辑

```go
// 支持 backend_path
if svc.BackendPath != "" {
    targetReq.URL.Path = singleJoiningSlash(svc.BackendPath, suffix)
} else {
    targetReq.URL.Path = suffix
}
```

## 📊 性能影响

### Session 刷新
- **开销**：每次请求增加 1 次 cookie 写入
- **影响**：可忽略（~1ms）

### Cookie 路径范围
- **开销**：无额外开销
- **影响**：无

### WebSocket 验证
- **开销**：每次 WebSocket 连接增加 1 次配置查询
- **影响**：可忽略（内存查询）
